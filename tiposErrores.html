Excepciones en Python

Excepciones Base

  • BaseException: Es la clase base para todas las excepciones. No deberías manejarla directamente.
  • Exception: Clase base para la mayoría de las excepciones. Se suele usar para capturar cualquier excepción que herede de ella.

Excepciones del Sistema

  • SystemExit: Se lanza cuando se usa sys.exit() para salir del programa.
  • KeyboardInterrupt: Se lanza cuando el usuario interrumpe la ejecución del programa (generalmente pulsando Ctrl+C).
  • GeneratorExit: Se lanza cuando un generador o coroutine finaliza.

Excepciones Estándar Más Comunes

ArithmeticError

  • ZeroDivisionError: División por cero.
  • OverflowError: Un número es demasiado grande para ser representado.
  • FloatingPointError: Error en operaciones de punto flotante.

LookupError

  • IndexError: Cuando un índice está fuera del rango en una secuencia.
  • KeyError: Cuando una clave no existe en un diccionario.
  • ValueError: Se lanza cuando una función recibe un argumento del tipo correcto, pero con un valor inapropiado.
  • TypeError: Se lanza cuando una operación o función recibe un objeto de un tipo inapropiado.
  • AttributeError: Se lanza cuando se intenta acceder a un atributo que no existe en un objeto.
  • ImportError: Se lanza cuando una importación falla.
  • ModuleNotFoundError: Se lanza cuando no se encuentra el módulo que se intenta importar.
  • NameError: Se lanza cuando una variable o nombre no está definido.
  • UnboundLocalError: Se lanza cuando una variable local se refiere antes de ser asignada.
  • FileNotFoundError: Se lanza cuando un archivo o directorio no se encuentra.
  • IOError: Ocurre cuando hay un error en una operación de entrada/salida. En Python 3.x, está unificada con OSError.
  • OSError: Se lanza cuando ocurre un error relacionado con el sistema operativo (por ejemplo, fallos en archivos o procesos).
  • PermissionError: Ocurre cuando se intenta realizar una operación que requiere permisos que no están disponibles.
  • FileExistsError: Ocurre cuando se intenta crear un archivo que ya existe.
  • IsADirectoryError: Ocurre cuando se trata un directorio como si fuera un archivo.
  • RuntimeError: Ocurre cuando se detecta un error que no pertenece a ninguna otra categoría.
  • RecursionError: Se lanza cuando la recursión es demasiado profunda.
  • MemoryError: Se lanza cuando el programa no puede asignar memoria adicional.
  • StopIteration: Se lanza para indicar que un iterador no tiene más elementos.
  • StopAsyncIteration: Se lanza para indicar que un objeto asíncrono no tiene más elementos.
  • AssertionError: Ocurre cuando falla una declaración assert.

Excepciones Más Específicas

  • EOFError: Se lanza cuando se encuentra el final de un archivo de manera inesperada.
  • EnvironmentError: Clase base para excepciones relacionadas con el entorno. En versiones recientes de Python, está unificada con OSError.
  • IndentationError: Error relacionado con la indentación del código Python.
  • TabError: Se lanza cuando hay una mezcla inconsistente de tabulaciones y espacios en la indentación.
  • UnicodeError: Error relacionado con la codificación o decodificación de Unicode.
  • UnicodeEncodeError: Ocurre al intentar codificar caracteres Unicode en una codificación específica.
  • UnicodeDecodeError: Ocurre al intentar decodificar bytes en una codificación específica.
  • UnicodeTranslateError: Ocurre durante la traducción de un carácter Unicode.

Excepciones de Advertencia

  • Warning: Clase base para advertencias que no interrumpen el flujo normal del programa.
  • DeprecationWarning: Se emite cuando se utiliza una característica que está en desuso.
  • PendingDeprecationWarning: Similar a DeprecationWarning, pero para características que están marcadas para eliminación futura.
  • SyntaxWarning: Advertencia sobre una construcción de sintaxis dudosa.
  • RuntimeWarning: Advertencia para situaciones de ejecución dudosas.
  • UserWarning: Advertencia generada por el usuario.
  • FutureWarning: Advertencia de que una característica va a cambiar en futuras versiones de Python.
